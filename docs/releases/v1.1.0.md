# YUSPEC Unity v1.1.0

YUSPEC is a text-based gameplay rule layer for Unity.

## Highlights

- VS Code extension for `.yuspec` files with syntax highlighting, keyword completion, hover descriptions, go-to-definition, bracket matching, and save-time brace diagnostics
- Clickable Unity Console diagnostics in `filename(line,column): error YUSPEC: message` format
- Typed entity properties for `int`, `float`, `bool`, `string`, and `string[]`
- Typed arguments passed into `[YuspecAction]` C# bindings
- Static analysis for event handler cycles, repeated interval re-trigger loops, and unreachable states
- ScriptableObject binding through `from "Assets/..."` with guarded write-back via `[YuspecMutable]`
- Lightweight dialogue blocks with `line`, `choice`, `end`, and built-in `start_dialogue`
- FileSystemWatcher-based hot reload that preserves current entity property values and updates affected runtime registrations
- Unity EditMode tests passing in the validated local environment

## Install via UPM

Add this dependency to your Unity project's `Packages/manifest.json`:

```json
{
  "dependencies": {
    "com.yuspec.unity": "https://github.com/Fovane/yuspec.git?path=/unity/Packages/com.yuspec.unity#v1.1.0"
  }
}
```

## VS Code Extension

The extension source lives in:

```text
tools/vscode-yuspec
```

Validate and package it with:

```bash
cd tools/vscode-yuspec
npm install
npm run compile
npx @vscode/vsce package
```

## Known Limitations

- Public preview release; not yet battle-tested in large production projects
- State machine support is a working subset
- Scenario test support is a working subset
- Hot reload does not attempt live scene migration
- Advanced project-specific type inference is still intentionally limited
- Networking, multiplayer replication, visual node editing, shader authoring, and animation authoring are out of scope

## Upgrade Notes

Existing users of v1.0.1 should update their Unity manifest dependency from `#v1.0.1` to `#v1.1.0`.
