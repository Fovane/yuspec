# YUSPEC Unity v1.1.1

YUSPEC is a text-based gameplay rule layer for Unity.

This patch release fixes the TopDownDungeon showcase behavior in the Unity package.

## Highlights

- Goblin chase now moves the goblin toward the player in the TopDownDungeon demo.
- Goblin attack damage is gated by distance, so the player only takes goblin damage while inside attack range.
- Boss damage no longer starts when the scene starts; the boss remains dormant until the player enters the boss room.
- `move_towards` validation now understands the runtime-injected actor parameter used by behavior state machines.
- Added Unity EditMode regression tests for goblin movement and dormant boss damage gating.

## Install via UPM

Add this dependency to your Unity project's `Packages/manifest.json`:

```json
{
  "dependencies": {
    "com.yuspec.unity": "https://github.com/Fovane/yuspec.git?path=/unity/Packages/com.yuspec.unity#v1.1.1"
  }
}
```

## Tested With

- Unity 6000.3.8f1
- Unity EditMode tests: 19 passed, 0 failed, 0 skipped in the validated local environment

## Upgrade Notes

Existing users of v1.1.0 should update their Unity manifest dependency from `#v1.1.0` to `#v1.1.1`.

If you download from GitHub Releases, use the `v1.1.1` release assets or source archive. The older `v1.1.0` release does not include the TopDownDungeon chase and idle-damage fixes.

## Known Limitations

- Public preview release; not yet battle-tested in large production projects
- State machine support is a working subset
- Scenario test support is a working subset
- Hot reload does not attempt live scene migration
- Advanced project-specific type inference is still intentionally limited
- Networking, multiplayer replication, visual node editing, shader authoring, and animation authoring are out of scope
